## 3. 常见问题与原因

| 问题现象                                                                  | 根本原因                                              |
| ------------------------------------------------------------------------- | ----------------------------------------------------- |
| `Unable to open/read ui device`                                         | `--add-data` 路径写错，或代码里没写对资源路径       |
| `ModuleNotFoundError`                                                   | PyInstaller 未识别动态导入，需要加`--hidden-import` |
| `qt.qpa.platform` / `Could not find the Qt platform plugin "windows"` | `plugins/platforms` 未正确打包                      |
| 运行时报错/闪退                                                           | `resource_path()` 写错、路径分隔符错误、exe 被占用  |

### 常见烧包原因

- **资源未打包**：ini、图片、ui 文件默认忽略，必须用 `--add-data` 指定。
- **隐藏控制台**：`--windowed` 时看不到错误，调试阶段一定要先用 `--console`。
- **Windows 路径问题**：
  - 用分号 `;` 分隔（Linux/Mac 用冒号 `:`）。
  - 反斜杠 `\` 在 shell 中可能被转义，必要时用引号包裹。
- **exe 被占用**：打包前先关闭程序，或 `taskkill /F /IM 程序名.exe`。

---

## 4. 打包流程

1. 确定项目结构，识别所有外部资源。
2. 在代码里加入 `resource_path()` 兼容开发/打包路径。
3. 执行打包命令，优先用 `--console` 调试。
4. 验证 `dist/程序名/` 目录和 `_internal` 资源。

---

## 5. 参数详解

| 参数                          | 作用                                           |
| ----------------------------- | ---------------------------------------------- |
| `--onedir`                  | 单文件夹模式：exe + 依赖目录，启动快，便于调试 |
| `--onefile`                 | 单文件模式：一个 exe，启动慢（需解压）         |
| `--name <name>`             | 指定生成的 exe 名称                            |
| `--console`                 | 显示控制台（调试用）                           |
| `--windowed`                | 隐藏控制台（GUI 发布用）                       |
| `--icon=logo.ico`           | 设置 exe 图标                                  |
| `--add-data "<src>;<dest>"` | 打包非 Python 资源文件                         |
| `--hidden-import=<module>`  | 手动指定隐藏导入模块                           |
| `--clean`                   | 清理缓存重新打包                               |
| `--paths <dir>`             | 添加模块搜索路径                               |
| `<script.py>`               | 主程序入口文件                                 |

## 6. 处理资源文件

资源文件包括：`.ui`、`.qml`、`.ico`、`.png`、`.jpg`、`.json`、`.csv` 等。

### 命令行方式

```bash
pyinstaller --add-data "ui/*.ui;ui" --add-data "img/logo.png;img" app.py
```

> Windows 使用分号；目的是把源码目录的资源复制到 `_internal` 的对应目录。

### .spec 方式

```python
datas=[
    ('ui/mainwindow.ui', 'ui'),
    ('data/config.json', 'data'),
]
```

## 7. 处理可视化界面资源

### Qt 界面文件（.ui）

```python
import os, sys

def resource_path(relative_path):
    try:
        base_path = sys._MEIPASS
    except AttributeError:
        base_path = os.path.dirname(os.path.abspath(__file__))
    return os.path.join(base_path, relative_path)

ui = loader.load(resource_path('ui/mainwindow.ui'))
```

### Qt 资源文件（.qrc 编译后的 .py）

```python
# 推荐用 resource_path + QPixmap
logo_path = resource_path('img/logo.png')
pixmap = QPixmap(logo_path)
```

## 8. 必须牢记

- 代码里所有外部文件读取都通过 `resource_path()`。
- 所有外部资源都通过 `--add-data` 打包。
- `_internal` 目录不能删。
- Windows 用分号，Linux/Mac 用冒号。
- 先 `--console` 调通，再 `--windowed` 发布。

## 10. 常见错误对照表

| 错误信息                                        | 原因                | 解决方案                                          |
| ----------------------------------------------- | ------------------- | ------------------------------------------------- |
| `Unable to open/read ui device`               | UI 未打包或路径写错 | `--add-data "ui/*.ui;ui"` + `resource_path()` |
| `ModuleNotFoundError`                         | 动态导入未识别      | 加`--hidden-import`                             |
| `qt.qpa.platform plugin "windows" not loaded` | 平台插件缺失        | 确保`plugins/platforms` 存在                    |
| `Failed to execute script`                    | 运行时错误          | 改用`--console` 查看详细错误                    |

## 11. 标准打包后文件夹结构

### 实际结构示例

```
MyApp/
├── MyApp.exe                ← 主程序入口
├── logo.png                 ← 额外资源文件（如有 --add-data）
└── _internal/               ← 依赖目录（PyInstaller 自动生成）
    ├── python3.dll
    ├── python314.dll
    ├── VCRUNTIME140.dll
    ├── Qt6Core.dll / Qt6Gui.dll / Qt6Widgets.dll
    ├── PySide6/             ← Qt 框架核心文件
    ├── shiboken6/           ← Qt 绑定库
    ├── certifi/
    ├── charset_normalizer/
    ├── setuptools/
    ├── ui/                  ← 你的 .ui 文件（如果用了 --add-data）
    │   └── mainwindow.ui
    ├── img/                 ← 图片资源（如果用了 --add-data）
    │   └── logo.png
    ├── data/                ← 数据文件（如果用了 --add-data）
    │   └── config.json
    ├── base_library.zip
    ├── _bz2.pyd
    ├── _ctypes.pyd
    └── ...（其他 DLL 和 pyd 文件）
```

### 常见情况与问题

1. **文件大小**：exe 本身几 MB，整个文件夹 50-100 MB，这是正常的
2. **`_internal` 目录不能删除**：exe 运行依赖它
3. **杀毒软件误报**：PyInstaller 打包的无签名 exe 容易被误报，可加白名单
4. **exe 被占用**：打包前需关闭运行中的程序，或 `taskkill /F /IM 程序名.exe`
5. **路径映射**：打包后路径通过 `sys._MEIPASS` 映射到 `_internal`，必须使用 `resource_path()`
6. **更新程序**：建议完整重新打包，避免版本不一致

## 12. 推荐工作流

### 调试打包

```bash
pyinstaller --onedir --name MyApp --console \
    --add-data "ui/*.ui;ui" \
    --add-data "data/*.json;data" \
    --hidden-import=PySide6.QtXml \
    --paths . \
    main.py
```

### 正式打包

```bash
rd /s /q build dist
pyinstaller --onedir --name MyApp --windowed --clean \
    --add-data "ui/*.ui;ui" \
    --add-data "data/*.json;data" \
    --hidden-import=PySide6.QtXml \
    --paths . \
    main.py
```

### 单文件模式

```bash
pyinstaller --onefile --windowed --clean \
    --add-data "ui/*.ui;ui" \
    --hidden-import=PySide6.QtXml \
    main.py
```

## 13. 自定义输出文件夹结构

如果你想自定义打包后的目录结构，有几种方法：

### 方法一：使用 spec 文件（推荐）

编辑 `.spec` 文件，修改 `datas` 和 `name`：

```python
# -*- mode: python ; coding: utf-8 -*-

a = Analysis(
    ['main.py'],
    pathex=['.'],
    binaries=[],
    datas=[
        ('ui/mainwindow.ui', 'ui'),           # 源路径, 目标相对路径
        ('data/config.json', 'config'),        # 可自定义子目录名
        ('img/logo.png', 'assets/images'),     # 多级目录
        ('README.md', '.'),                    # 直接放根目录
    ],
    hiddenimports=['PySide6.QtXml'],
    ...
)

# 修改输出目录名
exe = EXE(
    pyz, a.scripts, [],
    exclude_binaries=True,
    name='我的自定义名称',    # 修改 exe 名称
    ...
)

coll = COLLECT(
    exe, a.binaries, a.datas,
    strip=False, upx=True,
    name='我的自定义文件夹名',    # 修改输出文件夹名
)
```

执行：
```bash
pyinstaller --clean MyApp.spec
```

### 方法二：使用 `--distpath` 和 `--workpath`

```bash
pyinstaller --onedir \
    --name "MyApp" \
    --distpath "D:/发布/程序" \      # 指定 dist 输出目录
    --workpath "D:/构建/temp" \      # 指定 build 临时目录
    --specpath "D:/构建/spec" \      # 指定 spec 文件目录
    --add-data "ui/*.ui;ui" \
    main.py
```

### 方法三：手动整理打包结果

如果只是微调，可以直接复制打包后的文件夹再手动整理：

```bash
# 1. 正常打包
pyinstaller --onedir --name MyApp main.py

# 2. 复制到目标目录
xcopy "dist/MyApp" "D:/发布/MyApp_v1.0" /E /I

# 3. 手动删除不需要的文件
# 比如删除 .spec、build 目录、warn 文件等
```

### 方法四：使用脚本自动整理

创建 `organize.py`：

```python
import shutil
import os

# 定义目标结构
structure = {
    'MyApp.exe': 'MyApp.exe',
    '_internal/ui/mainwindow.ui': 'ui/mainwindow.ui',
    '_internal/img/logo.png': 'assets/logo.png',
    '_internal/data/config.json': 'config/config.json',
}

src_dir = 'dist/MyApp'
dst_dir = 'D:/发布/MyApp_v1.0'

# 创建目标目录
for dst in structure.values():
    os.makedirs(os.path.dirname(os.path.join(dst_dir, dst)), exist_ok=True)

# 复制文件
for src, dst in structure.items():
    shutil.copy(os.path.join(src_dir, src), os.path.join(dst_dir, dst))

# 复制 _internal 目录
shutil.copytree(os.path.join(src_dir, '_internal'), os.path.join(dst_dir, '_internal'))
```

### 实际示例：输出到子目录

假设项目结构：
```
项目/
├── main.py
├── ui/
├── dist/                    ← PyInstaller 默认输出
└── 发布/                    ← 你想放的位置
    └── v1.0/
        ├── MyApp.exe
        └── ...
```

命令：
```bash
pyinstaller --onedir --name MyApp \
    --distpath "发布/v1.0" \
    --add-data "ui/*.ui;ui" \
    main.py
```

### 注意事项

1. **`.spec` 路径是相对于执行目录的**：确保 `pathex=['.']` 或绝对路径
2. **`_internal` 必须和 exe 在同一目录**：不能拆散
3. **资源路径要对应**：代码里 `resource_path('ui/xxx.ui')` 要和 `--add-data "ui/xxx.ui;ui"` 一致
4. **清理旧文件**：重打包前删除旧的 `dist/` 和 `build/` 目录

### 推荐做法

- 用 `.spec` 文件管理复杂的打包配置
- 把 `.spec` 文件提交到版本控制
- 用 `--distpath` 控制最终输出位置
- 保持 `_internal` 完整性，不要手动拆分

## 14. 同一文件夹下多程序分别打包注意事项

如果一个文件夹下有多个 Python 程序，分别打包时需要注意：

### 核心原则

- **每个程序独立打包**，一次只打包一个
- **命名互不相同**，避免输出冲突
- **输出目录分开**，保持整洁
- **资源文件不冲突**，注意同名问题

### 具体操作

**1. 使用不同的 `--name` 和 `--distpath`**

```bash
# 程序 A
pyinstaller --onedir --name "工具A" --distpath "dist/工具A" \
    --add-data "ui_a/*.ui;ui" \
    program_a.py

# 程序 B
pyinstaller --onedir --name "工具B" --distpath "dist/工具B" \
    --add-data "ui_b/*.ui;ui" \
    program_b.py
```

**2. 输出目录结构示例**

```
python Qt/
├── first.py
├── 2.py
├── 3.py
├── 4.py
├── 5_httpclient.py
├── ui/
│   ├── stats.ui
│   └── 5.ui
└── dist/
    ├── 薪资统计/           ← first.py 的打包结果
    │   ├── 薪资统计.exe
    │   └── _internal/
    ├── HTTP客户端/        ← 5_httpclient.py 的打包结果
    │   ├── HTTP客户端.exe
    │   └── _internal/
    └── ...
```

**3. 资源文件同名问题**

如果两个程序有同名资源文件（如 `config.json`），打包时需加前缀区分：

```bash
# 程序 A
--add-data "a_config.json;config"

# 程序 B
--add-data "b_config.json;config"
```

代码中用 `resource_path('config/a_config.json')` 读取。

**4. 分别清理和打包**

```bash
# 清理所有 build/dist/spec
rd /s /q build dist spec

# 分别打包
pyinstaller --onedir --name "程序A" --distpath "dist/程序A" program_a.py
pyinstaller --onedir --name "程序B" --distpath "dist/程序B" program_b.py
```

**5. 使用独立 .spec 文件（推荐）**

创建 `spec/` 目录，为每个程序生成独立 spec：

```bash
pyinstaller --onedir --name "程序A" --specpath "spec" program_a.py
pyinstaller --onedir --name "程序B" --specpath "spec" program_b.py
```

后续更新直接用 spec：
```bash
pyinstaller --clean spec/程序A.spec
pyinstaller --clean spec/程序B.spec
```

**6. 批量打包脚本（可选）**

创建 `build_all.py`：

```python
import subprocess
import os

programs = [
    {"name": "薪资统计", "script": "first.py", "add_data": "ui/stats.ui;ui", "console": True},
    {"name": "HTTP客户端", "script": "5_httpclient.py", "add_data": "ui/5.ui;ui", "console": False},
]

for prog in programs:
    cmd = [
        "pyinstaller", "--onedir", "--name", prog["name"],
        "--distpath", f"dist/{prog['name']}",
        "--add-data", prog["add_data"],
    ]
    cmd.append("--console" if prog["console"] else "--windowed")
    cmd.append(prog["script"])
    
    print(f"正在打包 {prog['name']}...")
    subprocess.run(cmd)
    print(f"{prog['name']} 打包完成\n")
```

运行：`python build_all.py`

**7. 注意事项**

| 要点 | 说明 |
|------|------|
| **不要一次打包多个脚本** | PyInstaller 一次只能打包一个主程序 |
| **检查输出目录** | 确认每个程序的 `dist/程序名/` 都正确生成 |
| **测试每个 exe** | 分别双击测试，确保都能正常运行 |
| **独立版本管理** | 更新时只重新打包对应的程序 |
| **共享依赖** | 多个程序共享 `_internal` 是可能的，但新手不推荐 |
